## Overview

The **VerifiedMetrics** contract stores AI-verified story quality metrics attested by Chainlink CRE (Compute Runtime Environment). This is the **legacy version** that stores full metrics data on-chain for complete transparency.

**Status:** Legacy contract, maintained for backward compatibility. New deployments should use **PrivateVerifiedMetrics** for privacy-preserving attestation.

**Contract Address (Base Sepolia):** `0x052B52A4...`

## Architecture

### How It Works
1. User creates a story in eStory
2. Backend triggers CRE workflow (`POST /api/cre/trigger`)
3. Chainlink DON nodes fetch story, run AI analysis (Gemini), and reach consensus
4. DON nodes sign the report and send it to **KeystoneForwarder** (`0x82300bd7...`)
5. Forwarder verifies signatures and calls `onReport()` on this contract
6. Contract decodes and stores metrics on-chain
7. Buyers can view verified metrics before purchasing paywalled content

### Trust Model
- **Chainlink DON**: Decentralized network of nodes performs AI analysis in TEE (Trusted Execution Environment)
- **Cryptographic Proofs**: DON node signatures prove the analysis was performed correctly
- **On-Chain Attestation**: Metrics are immutable once written (cannot be edited by author or platform)

## Features

### Full Data On-Chain
All metrics are stored on-chain:
- Significance Score (0-100)
- Emotional Depth (1-5)
- Quality Score (0-100)
- Word Count
- Themes (string array)
- Attestation ID (unique report identifier)

### CRE Integration
Extends `ReceiverTemplate` interface to receive signed reports from Chainlink KeystoneForwarder.

### Public Verification
Anyone can verify metrics by calling `getMetrics(storyId)` on-chain.

<Accordion title="_processReport(bytes calldata report)">
  Internal function called by `ReceiverTemplate.onReport()` to decode and store CRE report data.

  **Visibility:** `internal`

  **Called By:** `onReport()` (from `ReceiverTemplate`), which is called by `KeystoneForwarder` after signature verification

  **Parameters:**
  - `report` (bytes) - ABI-encoded report data from CRE workflow

  **Report Encoding:**
  ```solidity
  abi.decode(
      report,
      (bytes32 storyId, address author, uint8 significanceScore,
       uint8 emotionalDepth, uint8 qualityScore, uint32 wordCount,
       string[] themes, bytes32 attestationId)
  )
  ```

  **Validation:**
  - `significanceScore <= 100`
  - `emotionalDepth >= 1 && emotionalDepth <= 5`
  - `qualityScore <= 100`

  **Implementation:**
  ```solidity
  function _processReport(bytes calldata report) internal override {
      (
          bytes32 storyId,
          address author,
          uint8 significanceScore,
          uint8 emotionalDepth,
          uint8 qualityScore,
          uint32 wordCount,
          string[] memory themes,
          bytes32 attestationId
      ) = abi.decode(
          report,
          (bytes32, address, uint8, uint8, uint8, uint32, string[], bytes32)
      );

      require(significanceScore <= 100, "Significance out of range");
      require(emotionalDepth >= 1 && emotionalDepth <= 5, "Emotional depth out of range");
      require(qualityScore <= 100, "Quality out of range");

      metrics[storyId] = Metrics({
          significanceScore: significanceScore,
          emotionalDepth: emotionalDepth,
          qualityScore: qualityScore,
          wordCount: wordCount,
          themes: themes,
          attestationId: attestationId,
          verifiedAt: block.timestamp,
          exists: true
      });

      emit MetricsVerified(
          storyId,
          author,
          significanceScore,
          emotionalDepth,
          qualityScore,
          wordCount,
          attestationId
      );
  }
  ```
</Accordion>

<Accordion title="getMetrics(bytes32 storyId)">
  Get all verified metrics for a story.

  **Access:** `public view`

  **Parameters:**
  - `storyId` (bytes32) - Story identifier (keccak256 hash)

  **Returns:**
  - `significanceScore` (uint8) - 0-100
  - `emotionalDepth` (uint8) - 1-5
  - `qualityScore` (uint8) - 0-100
  - `wordCount` (uint32)
  - `themes` (string[]) - Array of theme strings
  - `attestationId` (bytes32) - CRE report ID
  - `verifiedAt` (uint256) - Block timestamp

  **Requirements:**
  - Metrics must exist for the story (reverts with "Not verified" otherwise)

  ```solidity
  function getMetrics(bytes32 storyId) external view returns (
      uint8 significanceScore,
      uint8 emotionalDepth,
      uint8 qualityScore,
      uint32 wordCount,
      string[] memory themes,
      bytes32 attestationId,
      uint256 verifiedAt
  ) {
      require(metrics[storyId].exists, "Not verified");
      Metrics storage m = metrics[storyId];
      return (
          m.significanceScore,
          m.emotionalDepth,
          m.qualityScore,
          m.wordCount,
          m.themes,
          m.attestationId,
          m.verifiedAt
      );
  }
  ```

  **Usage Example:**
  ```typescript

  const metrics = await verifiedMetrics.getMetrics(storyId);
  
  console.log('Quality Score:', metrics.qualityScore);
  console.log('Themes:', metrics.themes);
  ```
</Accordion>

## Backward Compatibility

This contract is maintained for backward compatibility with early eStory deployments. New features should integrate with **PrivateVerifiedMetrics** instead.

### Migration Path
1. Deploy **PrivateVerifiedMetrics** contract
2. Update backend to trigger privacy-preserving CRE workflow
3. Maintain read access to legacy **VerifiedMetrics** for old stories
4. Display "Legacy Verified" badge for old contracts, "Verified" badge for new ones

### Differences from PrivateVerifiedMetrics

| Feature | VerifiedMetrics (Legacy) | PrivateVerifiedMetrics (Current) |
|---------|--------------------------|----------------------------------|
| Data on-chain | Full metrics (scores, themes, author) | Minimal (tier, threshold, hashes) |
| Privacy | Author address public | Author commitment hash (private) |
| Themes | Public string array | Off-chain (Supabase) |
| Use case | Full transparency | Privacy-preserving proof |

## Security Considerations

### Signature Verification
The `KeystoneForwarder` verifies DON node signatures before calling `onReport()`. This contract trusts the forwarder to perform signature verification correctly.

### Immutability
Once metrics are written, they cannot be updated. This prevents tampering but means errors in CRE reports are permanent.

### Privacy Trade-offs
All metrics are public on-chain. Authors cannot hide low scores or sensitive themes. Consider using **PrivateVerifiedMetrics** for privacy-sensitive deployments.

### Backend: Poll for Verification
```typescript
import { ethers } from 'ethers';
import { VERIFIED_METRICS_ADDRESS, VERIFIED_METRICS_ABI } from '@/lib/contracts';

const provider = new ethers.JsonRpcProvider(process.env.BASE_SEPOLIA_RPC);
const contract = new ethers.Contract(VERIFIED_METRICS_ADDRESS, VERIFIED_METRICS_ABI, provider);

async function waitForVerification(storyId: string, maxWaitMs = 60000) {
  const storyIdHash = ethers.keccak256(ethers.toUtf8Bytes(storyId));
  const startTime = Date.now();
  
  while (Date.now() - startTime < maxWaitMs) {
    const verified = await contract.isVerified(storyIdHash);
    if (verified) {
      const metrics = await contract.getMetrics(storyIdHash);
      return metrics;
    }
    await new Promise(resolve => setTimeout(resolve, 5000)); // Poll every 5s
  }
  
  throw new Error('Verification timeout');
}
```
