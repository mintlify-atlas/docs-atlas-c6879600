## Overview

The **PrivateVerifiedMetrics** contract provides privacy-preserving attestation of AI-verified story quality metrics using Chainlink CRE (Compute Runtime Environment). Unlike the legacy **VerifiedMetrics** contract, this version stores only **minimal cryptographic proofs** on-chain while keeping full metrics off-chain.

**Contract Address (Base Sepolia):** `0x158e08BC...`

## Privacy Design

### What's On-Chain (Public)
- **Quality Tier** (1-5): Coarse-grained quality indicator (not exact score)
- **Threshold Flag**: Boolean indicating if quality score >= 70
- **Metrics Hash**: `keccak256` hash of full metrics (provable off-chain)
- **Author Commitment**: `keccak256(authorWallet, storyId)` (not reversible)
- **Attestation ID**: CRE report identifier

### What's Off-Chain (Private)
- **Exact Scores**: Significance, emotional depth, quality (0-100)
- **Word Count**: Precise word count
- **Themes**: Extracted themes (e.g., ["growth", "loss", "hope"])
- **Author Address**: Stored in Supabase, visible only to author

### Why This Matters
- **Authors control disclosure**: Can choose to share exact scores or just show "Verified" badge
- **No theme leakage**: Sensitive themes (e.g., trauma, mental health) stay private
- **Provable integrity**: Anyone can verify metrics hash matches claimed data
- **Marketplace trust**: Buyers can see quality tier without revealing author identity

## Architecture

### Dual-Write Pattern
1. **On-Chain Write**: CRE workflow writes minimal proof to this contract via KeystoneForwarder
2. **Off-Chain Write**: CRE workflow posts full metrics to `/api/cre/callback` (stored in Supabase)
3. **Author-Filtered Reads**: `/api/cre/check` returns full metrics to author, proof-only to others

```
CRE Workflow → [On-Chain] PrivateVerifiedMetrics.sol (tier, threshold, hashes)
            → [Off-Chain] /api/cre/callback → Supabase (full metrics, author-only)
                                                    ↓
                                          /api/cre/check (author-filtered)
```

<Accordion title="getMetrics(bytes32 storyId)">

  **Parameters:**

  **Returns:**

  - `attestationId` (bytes32) - CRE report ID
  - `verifiedAt` (uint256) - Block timestamp

  **Requirements:**
  - Metrics must exist for the story (reverts with "Not verified" otherwise)

  ```solidity
  function getMetrics(bytes32 storyId) external view returns (
      bool meetsQualityThreshold,
      uint8 qualityTier,
      bytes32 metricsHash,
      bytes32 authorCommitment,
      bytes32 attestationId,
      uint256 verifiedAt

      require(metrics[storyId].exists, "Not verified");
      MinimalMetrics storage m = metrics[storyId];
      return (
          m.meetsQualityThreshold,
          m.qualityTier,
          m.metricsHash,
          m.authorCommitment,
          m.attestationId,
          m.verifiedAt
      );
  }
  ```

```typescript
async function processAnalysis(storyData, analysis) {
  // 1. Compute quality tier (coarse-grained)
  const qualityTier = Math.ceil(analysis.qualityScore / 20); // 1-5
  const meetsThreshold = analysis.qualityScore >= 70;
  
  // 2. Compute author commitment (hide raw address)
  const authorCommitment = ethers.keccak256(
    ethers.solidityPacked(['address', 'bytes32'], [authorAddress, storyId])
  );
  
  // 3. Compute metrics hash (provable integrity)
  const metricsHash = ethers.keccak256(
    ethers.AbiCoder.defaultAbiCoder().encode(
      ['uint8', 'uint8', 'uint8', 'uint32', 'string[]', 'bytes32'],
      [
        analysis.significanceScore,
        analysis.emotionalDepth,
        analysis.qualityScore,
        analysis.wordCount,
        analysis.themes,
        attestationId
      ]
    )
  );
  
  // 4. Encode minimal report for on-chain storage
  const onChainReport = encodeAbiParameters([
    { type: 'bytes32', name: 'storyId' },
    { type: 'bytes32', name: 'authorCommitment' },
    { type: 'bool', name: 'meetsQualityThreshold' },
    { type: 'uint8', name: 'qualityTier' },
    { type: 'bytes32', name: 'metricsHash' },
    { type: 'bytes32', name: 'attestationId' }
  ], [storyId, authorCommitment, meetsThreshold, qualityTier, metricsHash, attestationId]);
  
  // 5. Send minimal report to KeystoneForwarder (on-chain)
  await sendReport(onChainReport);
  
  // 6. Send full metrics to backend (off-chain)
  await fetch('https://estory.app/api/cre/callback', {
    method: 'POST',
    body: JSON.stringify({
      storyId,
      authorAddress, // Raw address (backend only)
      significanceScore: analysis.significanceScore,
      emotionalDepth: analysis.emotionalDepth,
      qualityScore: analysis.qualityScore,
      wordCount: analysis.wordCount,
      themes: analysis.themes,
      attestationId,
      metricsHash // For integrity verification
    })
  });
}
```

### 3. Author-Filtered Reads
```typescript
// GET /api/cre/check?storyId=story-123
export async function GET(req: NextRequest) {
  const authResult = await validateAuthOrReject(req);
  if (isAuthError(authResult)) return authResult;
  const userId = authResult;
  
  const storyId = req.nextUrl.searchParams.get('storyId');
  
  // 1. Get on-chain proof (public)
  const storyIdHash = ethers.keccak256(ethers.toUtf8Bytes(storyId));
  const onChainMetrics = await privateVerifiedMetrics.getMetrics(storyIdHash);
  
  // 2. Check if user is author
  const { data: story } = await supabase
    .from('stories')
    .select('user_id')
    .eq('id', storyId)
    .single();
  
  const isAuthor = story?.user_id === userId;
  
  if (isAuthor) {
    // 3. Return full metrics from Supabase (author only)
    const { data: fullMetrics } = await supabase
      .from('verified_metrics')
      .select('*')
      .eq('story_id', storyId)
      .single();
    
    return NextResponse.json({
      proof: onChainMetrics,
      fullMetrics,
      isAuthor: true
    });
  } else {
    // 4. Return only on-chain proof (non-authors)
    return NextResponse.json({
      proof: onChainMetrics,
      isAuthor: false
    });
  }
}
```

## Privacy Guarantees

### What Attackers CANNOT Learn
1. **Author Identity**: Only commitment hash on-chain (not reversible without knowing both address and storyId)
2. **Exact Scores**: Only tier (1-5) and threshold flag (>=70) on-chain
3. **Themes**: Stored off-chain in Supabase, visible only to author
4. **Word Count**: Not on-chain

### What Attackers CAN Learn
1. **Story Verified**: `isVerified(storyId)` returns true
2. **Quality Tier**: 1-5 (coarse-grained)
3. **Meets Threshold**: Boolean (quality >= 70)
4. **Verification Time**: Block timestamp

### Proof Verification
Anyone can verify claimed metrics without revealing them:
```typescript
// Author shares metrics off-chain
const claimedMetrics = { qualityScore: 85, themes: ['growth', 'hope'], ... };

// Verifier computes hash
const claimedHash = ethers.keccak256(
  ethers.AbiCoder.defaultAbiCoder().encode([...], claimedMetrics)
);

// Check on-chain
const valid = await privateVerifiedMetrics.verifyMetricsHash(storyId, claimedHash);
// valid === true without revealing metrics to the blockchain
```

## Security Considerations

### Author Commitment Security
The commitment hash `keccak256(abi.encodePacked(author, storyId))` is not reversible because:
- **High entropy**: Wallet addresses are 160-bit random values
- **Salted with storyId**: Each commitment is unique to story
- **Preimage resistance**: keccak256 is cryptographically secure

Attackers cannot brute-force author addresses from commitment hashes.

### Metrics Hash Integrity
The metrics hash binds the full metrics to the on-chain proof:
- **Tamper-proof**: Changing any field breaks the hash
- **Provable**: Authors can prove metrics without on-chain storage
- **Efficient**: Single bytes32 instead of full struct

### ReceiverTemplate Interface
Extends Chainlink's `ReceiverTemplate` to receive signed CRE reports:
- **Signature verification**: KeystoneForwarder verifies DON signatures
- **Replay protection**: Each report has unique attestationId
- **Access control**: Only forwarder can call `onReport()`
