## Overview

The **StoryNFT** contract enables users to mint NFTs representing story collections (books, anthologies) and awards weekly winner NFTs to top creators. Each NFT stores metadata on IPFS and supports ERC2981 royalties for secondary sales.

**Contract Address (Base Sepolia):** `0x6D37ebc5...`

## Features

### Dual Minting System
1. **Public Book Minting** - Any user can mint a book NFT for 0.001 ETH (prevents spam)
2. **Weekly Winner Minting** - Admin-only free minting for contest winners

### ERC721 Standard
Full compliance with ERC721 NFT standard:
- Unique token IDs (sequential, starting from 1)
- Safe transfers with `_safeMint` (prevents accidental burns to contracts)
- Enumeration via OpenZeppelin extensions

### IPFS Metadata Storage
Each NFT has a `tokenURI` pointing to IPFS metadata (cover art, title, description, story list).

### ERC2981 Royalties
- Default royalty: **5%** (500 basis points)
- Applied to secondary sales on compatible marketplaces (OpenSea, Rarible, etc.)
- Royalty receiver can be updated by admin

### Access Control
- **DEFAULT_ADMIN_ROLE**: Can update mint fee, set royalties, withdraw fees
- **MINTER_ROLE**: Can mint weekly winner NFTs for free

## Functions

<Accordion title="mintWeeklyWinner(address winner, string memory uri)">
  Mint a free NFT to a weekly contest winner. Only callable by MINTER_ROLE holders.

  **Access:** `onlyRole(MINTER_ROLE)`

  **Parameters:**
  - `winner` (address) - Recipient wallet address
  - `uri` (string) - IPFS URI for NFT metadata (e.g., `ipfs://QmXxx...`)

  **Implementation:**
  ```solidity
  function mintWeeklyWinner(address winner, string memory uri) 
      external 
      onlyRole(MINTER_ROLE) 
  {
      uint256 tokenId = ++_nextTokenId;
      _safeMint(winner, tokenId);
      _setTokenURI(tokenId, uri);
      emit NFTMinted(tokenId, winner, uri, "WEEKLY_WINNER");
  }
  ```

  **Usage Example:**
  ```typescript
  // Backend awards weekly winner
  const metadataURI = 'ipfs://QmWeeklyWinnerMetadata123';
  await storyNFT.mintWeeklyWinner(winnerAddress, metadataURI);
  ```
</Accordion>

## Royalty Mechanics

### How ERC2981 Works
1. User lists NFT on marketplace (e.g., OpenSea)
2. Buyer purchases NFT for 1 ETH
3. Marketplace calls `royaltyInfo(tokenId, salePrice)` on contract
4. Contract returns `(receiver, royaltyAmount)` (e.g., treasury address, 0.05 ETH for 5%)
5. Marketplace sends 0.05 ETH to receiver, 0.95 ETH to seller

### Default Royalty
Set at deployment to 5% (500 basis points) with `defaultAdmin` as receiver:
```solidity
_setDefaultRoyalty(defaultAdmin, 500);
```

### Marketplace Compatibility
Supported by:
- OpenSea (automatic royalty enforcement)
- Rarible
- LooksRare
- Foundation
- Any marketplace implementing ERC2981

## Minting Workflows

### Public Book Minting Flow
1. User compiles story collection in eStory app
2. Frontend uploads metadata to IPFS via Pinata:
   ```typescript
   const metadata = {
     name: bookTitle,
     description: bookDescription,
     image: coverImageIPFS,
     attributes: [/* traits */]
   };
   const { IpfsHash } = await pinata.pinJSON(metadata);
   const uri = `ipfs://${IpfsHash}`;
   ```
3. User calls `mintBook(uri)` with 0.001 ETH
4. NFT is minted and appears in user's wallet
5. Backend indexes `NFTMinted` event and displays in user's profile

### Weekly Winner Minting Flow
1. Backend runs weekly contest query (most likes, highest engagement, etc.)
2. Winner is selected
3. Backend generates special winner metadata and uploads to IPFS
4. Admin wallet calls `mintWeeklyWinner(winnerAddress, uri)`
5. Winner receives NFT for free
6. Winner is notified via email/notification

## Security Considerations

### Mint Fee Protection
The 0.001 ETH mint fee prevents spam minting attacks. Without a fee, an attacker could mint millions of NFTs to bloat storage.

### Safe Minting
Uses `_safeMint` instead of `_mint` to prevent accidental transfers to contracts that don't support ERC721:
```solidity
_safeMint(winner, tokenId); // Calls onERC721Received if recipient is a contract
```

### Withdrawal Safety
Uses low-level `call` instead of `transfer` to avoid gas limit issues:
```solidity
(bool success, ) = payable(msg.sender).call{value: balance}("");
require(success, "Withdrawal failed");
```

### Royalty Bounds
ERC2981 allows royalties up to 10,000 basis points (100%). The admin should never set royalties above 10% to avoid marketplace rejection.

## Integration Examples

### Frontend: Mint Book NFT
```typescript
import { useStoryNFT } from '@/app/hooks/useStoryNFT';
import { uploadMetadataToIPFS } from '@/lib/ipfs';
import { ethers } from 'ethers';

function MintBookFlow({ bookData }) {
  const { mintBook } = useStoryNFT();
  const [loading, setLoading] = useState(false);
  
  const handleMint = async () => {
    setLoading(true);
    try {
      // 1. Upload metadata to IPFS
      const metadataURI = await uploadMetadataToIPFS({
        name: bookData.title,
        description: bookData.description,
        image: bookData.coverImageIPFS,
        attributes: bookData.attributes
      });
      
      // 2. Mint NFT
      const mintFee = ethers.parseEther("0.001");
      const tx = await mintBook(metadataURI, { value: mintFee });
      await tx.wait();
      
      toast.success('Book NFT minted!');
    } catch (error) {
      console.error('Mint failed:', error);
      toast.error('Failed to mint NFT');
    } finally {
      setLoading(false);
    }
  };
  
  return (
    <button onClick={handleMint} disabled={loading}>
      {loading ? 'Minting...' : 'Mint Book NFT (0.001 ETH)'}
    </button>
  );
}
```

### Backend: Award Weekly Winner
```typescript
import { ethers } from 'ethers';
import { STORY_NFT_ADDRESS, STORY_NFT_ABI } from '@/lib/contracts';
import { uploadMetadataToIPFS } from '@/lib/ipfs';

const provider = new ethers.JsonRpcProvider(process.env.BASE_SEPOLIA_RPC);
const signer = new ethers.Wallet(process.env.MINTER_PRIVATE_KEY, provider);
const nft = new ethers.Contract(STORY_NFT_ADDRESS, STORY_NFT_ABI, signer);

async function awardWeeklyWinner(winnerAddress: string, weekNumber: number) {
  // 1. Generate winner metadata
  const metadata = {
    name: `eStory Weekly Winner - Week ${weekNumber}`,
    description: `Top creator of week ${weekNumber} on eStory`,
    image: 'ipfs://QmWinnerBadgeImage',
    attributes: [
      { trait_type: 'Week', value: weekNumber },
      { trait_type: 'Award', value: 'Weekly Winner' },
      { trait_type: 'Year', value: '2026' }
    ]
  };
  
  // 2. Upload to IPFS
  const uri = await uploadMetadataToIPFS(metadata);
  
  // 3. Mint NFT
  const tx = await nft.mintWeeklyWinner(winnerAddress, uri);
  await tx.wait();
  
  console.log(`Awarded weekly winner NFT to ${winnerAddress}`);
  return tx.hash;
}
```

contract StoryNFT is ERC721URIStorage, AccessControl, ERC2981 {
    bytes32 public constant MINTER_ROLE = keccak256("MINTER_ROLE");
    uint256 private _nextTokenId;

    uint256 public mintFee = 0.001 ether; // Small fee to prevent spam

    event NFTMinted(uint256 indexed tokenId, address indexed recipient, string uri, string collectionType);

    constructor(address defaultAdmin) ERC721("EStory Collections", "ESTORY") {
        _grantRole(DEFAULT_ADMIN_ROLE, defaultAdmin);
        _grantRole(MINTER_ROLE, defaultAdmin);

        // Set default royalty to 5% (500 basis points)
        _setDefaultRoyalty(defaultAdmin, 500);
    }
}
