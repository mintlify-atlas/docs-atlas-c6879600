## Overview

Generates a thoughtful, empathetic weekly reflection by analyzing the user's journal entries from the past 7 days. Uses Google Gemini 2.5 Flash to identify patterns, themes, and provide supportive insights. Limited to one reflection per week.

## Authentication

Requires Bearer token authentication (future implementation). Currently validates via request body parameters.

```bash
Authorization: Bearer YOUR_SUPABASE_JWT
```

## Endpoints

### GET - Fetch Reflections

Retrieve existing reflections for a user.

<ParamField query="userId" type="string" required>
  UUID of the user whose reflections to fetch.
</ParamField>

<ParamField query="limit" type="number" default="5">
  Maximum number of reflections to return.
</ParamField>

### POST - Generate Reflection

Create a new weekly reflection.

<ParamField body="userId" type="string" required>
  UUID of the user requesting the reflection.
</ParamField>

<ParamField body="userWallet" type="string" required>
  Ethereum wallet address of the user (for future Web3 features).
</ParamField>

## Limits

- **Generation frequency:** 1 reflection per week (7-day cooldown)
- **Stories analyzed:** Maximum 20 most recent stories from past 7 days
- **Canonical weight:** Stories marked as canonical count 2x
- **Week definition:** Sunday to Saturday
- **Reflection length:** 2-3 paragraphs (max 500 words)
- **Model:** Gemini 2.5 Flash

## Reflection Guidelines

The AI reflection follows these principles:

1. **Warm and encouraging:** Uses a compassionate life coach tone, not clinical
2. **Pattern-focused:** Identifies themes and patterns across multiple entries
3. **Non-prescriptive:** Offers perspective without telling users what to do
4. **Authentic:** Acknowledges sparse entries gently without judgment
5. **Forward-looking:** Encourages continued journaling and self-reflection

## Empty Week Handling

If the user has no journal entries in the past 7 days, the API generates an encouraging message about the value of journaling instead of a pattern-based reflection.

## Canonical Story Weighting

Stories marked as "canonical" (Important/Key Moments) receive 2x weight in the analysis:

- **Regular story:** Weight = 1.0
- **Canonical story:** Weight = 2.0
- The `canonical_weight` field sums all weights
- Canonical stories are emphasized in the prompt with `**KEY MOMENT**` markers

This ensures that significant life events have more influence on the reflection than routine entries.

## Database Tables

Stores generated reflections with metadata:

- `id` (UUID, primary key)
- `user_id` (UUID, foreign key → users)
- `user_wallet` (text)
- `reflection_text` (text)
- `stories_analyzed` (UUID[])
- `themes_identified` (text[])
- `dominant_tone` (text)
- `dominant_domain` (text)
- `week_start` (timestamp)
- `week_end` (timestamp)
- `canonical_weight` (numeric)
- `created_at` (timestamp)

## Rate Limiting

- **Weekly limit:** 1 reflection per user per week
- **Cooldown period:** 7 days from last generation
- **Week boundary:** Sunday 00:00:00 to Saturday 23:59:59
- **Enforcement:** Server-side check against existing reflections

## Use Cases

- **Weekly review:** Help users reflect on their week's experiences
- **Pattern discovery:** Identify emotional and thematic patterns
- **Progress tracking:** See how themes evolve over time
- **Motivation:** Encourage continued journaling with insights
- **Mental health:** Gain perspective on emotional states and triggers
