## Overview

Performs cognitive analysis on a story to extract structured metadata including themes, emotional tone, life domains, entities, and significance scores. Uses Google Gemini 2.5 Flash with retry logic and rate limiting.

## Authentication

Requires Bearer token authentication. User must own the story being analyzed.

```bash
Authorization: Bearer YOUR_SUPABASE_JWT
```

## Request

<ParamField body="storyId" type="string" required>
  UUID of the story to analyze. Must belong to the authenticated user.
</ParamField>

<ParamField body="storyText" type="string" required>
  The story content to analyze. Minimum 50 characters, maximum 10,000 characters. Longer content will be truncated.
</ParamField>

### Limits

- **Minimum content length:** 50 characters
- **Maximum content length:** 10,000 characters (truncated at word boundary)
- **Rate limit:** 30 requests per minute per story ID
- **Concurrent analysis:** One analysis per story at a time
- **Retry attempts:** Up to 3 attempts with exponential backoff

## Response

<ResponseField name="success" type="boolean">
  Indicates if the analysis completed successfully.
</ResponseField>

<ResponseField name="metadata" type="object">
  The extracted story metadata object.
  
  <Expandable title="metadata properties">
    
    <ResponseField name="story_id" type="string">
      UUID of the analyzed story.
    </ResponseField>
    
    <ResponseField name="themes" type="string[]">
      1-5 key themes extracted from the story (e.g., "growth", "loss", "connection").
    </ResponseField>
    
    <ResponseField name="emotional_tone" type="string">
      Dominant emotional tone. One of: `reflective`, `joyful`, `anxious`, `hopeful`, `melancholic`, `grateful`, `frustrated`, `peaceful`, `excited`, `uncertain`, `neutral`.
    </ResponseField>
    
    <ResponseField name="life_domain" type="string">
      Primary life domain. One of: `work`, `relationships`, `health`, `identity`, `growth`, `creativity`, `spirituality`, `family`, `adventure`, `learning`, `general`.
    </ResponseField>
    
    <ResponseField name="intensity_score" type="number">
      Emotional intensity score from 0.0 (calm) to 1.0 (highly charged).
    </ResponseField>
    
    <ResponseField name="significance_score" type="number">
      Life significance score from 0.0 (routine) to 1.0 (life-changing).
    </ResponseField>
    
    <ResponseField name="people_mentioned" type="string[]">
      Proper names of people mentioned in the story.
    </ResponseField>
    
    <ResponseField name="places_mentioned" type="string[]">
      Specific locations mentioned in the story.
    </ResponseField>
    
    <ResponseField name="time_references" type="string[]">
      Time references like "last summer", "when I was 12", "yesterday".
    </ResponseField>
    
    <ResponseField name="brief_insight" type="string">
      A compassionate one-sentence insight about the story's meaning.
    </ResponseField>
    
    <ResponseField name="updated_at" type="string">
      ISO 8601 timestamp of when metadata was generated.
    </ResponseField>
    
  </Expandable>
</ResponseField>

<ResponseField name="insight" type="string">
  The brief insight extracted from the analysis (same as `metadata.brief_insight`).
</ResponseField>

## Example

<CodeGroup>

```bash curl
curl -X POST https://your-domain.com/api/ai/analyze \
  -H "Authorization: Bearer YOUR_SUPABASE_JWT" \
  -H "Content-Type: application/json" \
  -d '{
    "storyId": "123e4567-e89b-12d3-a456-426614174000",
    "storyText": "Today I had an important conversation with my sister. We talked about our childhood and how much we've grown. It was emotional but also healing. I realized that family bonds are deeper than I thought."
  }'
```

```javascript JavaScript
const response = await fetch('/api/ai/analyze', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    storyId: '123e4567-e89b-12d3-a456-426614174000',
    storyText: 'Today I had an important conversation with my sister. We talked about our childhood and how much we\'ve grown. It was emotional but also healing. I realized that family bonds are deeper than I thought.'
  })
});

const { success, metadata, insight } = await response.json();
console.log('Metadata:', metadata);
console.log('Insight:', insight);
```

## Implementation Details

### Retry Logic

- **Maximum retries:** 3 attempts
- **Backoff strategy:** Exponential (1s, 2s, 4s)
- **Retry triggers:** Rate limits, timeouts, transient errors

### Content Processing

- Text longer than 10,000 characters is truncated at a word boundary
- Truncation is logged and indicated in metadata
- Response JSON is cleaned of markdown code blocks before parsing

### Validation

- `emotional_tone` must be one of the valid tones, defaults to `neutral`
- `life_domain` must be one of the valid domains, defaults to `general`
- Scores are clamped to [0.0, 1.0] range
- Arrays are limited to reasonable sizes (5 themes max)

### Database Storage

Metadata is upserted to `story_metadata` table with conflict resolution on `story_id`. The operation is atomic with proper error handling.

## Rate Limiting

- **AI endpoint limit:** 10 requests/minute (middleware)
- **Per-story limit:** 30 requests/minute (in-memory tracker)
- **Concurrent:** One analysis per story at a time

## Ownership Verification

The endpoint verifies that the authenticated user owns the story before performing analysis. This prevents unauthorized analysis of other users' private stories.

## Use Cases

- **Automatic tagging:** Extract themes for organization
- **Pattern discovery:** Identify emotional patterns over time
- **Search enhancement:** Enable semantic search by metadata
- **Insights generation:** Power weekly reflections and analytics
- **Privacy-preserving metrics:** Feed CRE verification with local analysis
