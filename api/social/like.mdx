---
title: 'Like Story'
description: 'Atomic toggle like/unlike for a story'
---

## Endpoint

<RequestExample>
```bash cURL
curl -X POST https://your-domain.com/api/social/like \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "storyId": "123e4567-e89b-12d3-a456-426614174000"
  }'
```
</RequestExample>

## Request

<ParamField body="storyId" type="string" required>
  The UUID of the story to like/unlike
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the operation was successful
</ResponseField>

<ResponseField name="data" type="object">
  Contains the like operation results
  
  <ResponseField name="data.storyId" type="string">
    The story ID that was liked/unliked
  </ResponseField>
  
  <ResponseField name="data.isLiked" type="boolean">
    Current like state after toggle (true = liked, false = unliked)
  </ResponseField>
  
  <ResponseField name="data.totalLikes" type="number">
    Total number of likes on the story
  </ResponseField>
  
  <ResponseField name="data.timestamp" type="string">
    ISO 8601 timestamp of the operation
  </ResponseField>
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable success message ("Story liked!" or "Story unliked.")
</ResponseField>

<ResponseExample>
```json Response
{
  "success": true,
  "data": {
    "storyId": "123e4567-e89b-12d3-a456-426614174000",
    "isLiked": true,
    "totalLikes": 42,
    "timestamp": "2026-03-03T23:15:30.123Z"
  },
  "message": "Story liked!"
}
```
</ResponseExample>

<Note>
  This endpoint performs an **atomic toggle**: if the story is already liked, it will be unliked, and vice versa. The operation is idempotent and thread-safe.
</Note>

## Behavior

- **Like**: Inserts a new like record and increments the story's like count
- **Unlike**: Deletes the existing like record and decrements the story's like count (floor at 0)
- Creates a notification for the story author when liked (unless you're liking your own story)
- Requires authentication via Bearer token

## Error Responses

| Status | Error | Description |
|--------|-------|-------------|
| 400 | Story ID is required | Missing `storyId` in request body |
| 401 | Unauthorized | Invalid or missing Bearer token |
| 500 | Failed to like/unlike | Database operation failed |
