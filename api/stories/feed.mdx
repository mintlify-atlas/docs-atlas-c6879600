---
title: "Get Stories Feed"
api: "GET /api/stories/feed"
description: "Fetch public stories with author information and pagination"
authMethod: "none"
---

Returns a feed of public stories ordered by creation date (newest first). This endpoint bypasses Row Level Security (RLS) to show public content without authentication.

## Query Parameters

<ParamField query="limit" type="number" optional default="20">
  Maximum number of stories to return (not currently enforced in implementation)
</ParamField>

<ParamField query="offset" type="number" optional default="0">
  Number of stories to skip for pagination (not currently enforced in implementation)
</ParamField>

<Info>
The current implementation returns all public stories. Pagination parameters are documented for future use.
</Info>

## Example Request

```bash cURL
curl -X GET "https://estory.app/api/stories/feed?limit=20&offset=0"
```

## Notes

- Stories are ordered by `created_at` descending (newest first)
- This endpoint bypasses Supabase RLS using the admin client
- Author information is joined via the `stories_author_wallet_fkey` foreign key
- Empty arrays are returned if no public stories exist
