---
title: "Save Story"
api: "POST /api/journal/save"
description: "Create a new story with optional audio and trigger AI analysis"
authMethod: "bearer"
---

Saves a new journal entry/story to the database. Automatically triggers background AI analysis and CRE (Chainlink Compute Runtime Environment) verification workflows.

## Request Body

<ParamField body="title" type="string" optional>
  Story title (defaults to "Journal Entry [date]" if not provided)
</ParamField>

<ParamField body="content" type="string" required>
  Story text content (cannot be empty or whitespace only)
</ParamField>

<ParamField body="audioUrl" type="string" optional>
  IPFS or storage URL for audio recording
</ParamField>

<ParamField body="mood" type="string" optional>
  Mood/emotion indicator (defaults to `"neutral"`)
</ParamField>

<ParamField body="tags" type="array" optional>
  Array of tag strings for categorization
</ParamField>

<ParamField body="hasAudio" type="boolean" optional>
  Whether the story has an associated audio file
</ParamField>

## Background Processes

Saving a story triggers two asynchronous workflows:

### 1. AI Analysis (`/api/ai/analyze`)
- Extracts themes, entities, emotional tone
- Populates `story_metadata` table
- Non-blocking (fire-and-forget)

### 2. CRE Verification (`/api/cre/trigger`)
- Chainlink DON attests to AI analysis integrity
- Writes privacy-preserving proof to `PrivateVerifiedMetrics` contract
- Stores full metrics off-chain (Supabase)
- Non-blocking (fire-and-forget)

<Info>
Both background processes are non-critical. If they fail, they can be retried later without affecting the story creation.
</Info>
