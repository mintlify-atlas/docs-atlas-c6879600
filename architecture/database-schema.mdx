# Database Schema

eStory uses Supabase (PostgreSQL) for cloud storage and Dexie.js (IndexedDB) for client-side encrypted local storage.

## Core Tables

### users

<Accordion title="Schema" icon="user">
  ```sql
  CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    wallet_address TEXT UNIQUE,
    google_id TEXT UNIQUE,
    google_email TEXT,
    google_avatar TEXT,
    auth_provider TEXT DEFAULT 'wallet', -- 'wallet' | 'google' | 'both'
    name TEXT,
    username TEXT UNIQUE,
    bio TEXT,
    avatar TEXT,
    location TEXT,
    website TEXT,
    badges TEXT[] DEFAULT '{}',
    streak_count INTEGER DEFAULT 0,
    total_stories INTEGER DEFAULT 0,
    total_likes INTEGER DEFAULT 0,
    followers_count INTEGER DEFAULT 0,
    following_count INTEGER DEFAULT 0,
    monthly_goal INTEGER DEFAULT 10,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
  );
  
  CREATE INDEX idx_users_wallet ON users(wallet_address);
  CREATE INDEX idx_users_google ON users(google_id);
  CREATE INDEX idx_users_username ON users(username);
  ```
  
  **Unique Constraints:**
  - `wallet_address` — Blockchain identity
  - `google_id` — Google OAuth ID
  - `username` — Display username
</Accordion>

<Accordion title="RLS Policies" icon="shield">
  ```sql
  -- Anyone can view public user profiles
  CREATE POLICY "Public profiles are viewable by everyone"
  ON users FOR SELECT
  USING (true);
  
  -- Users can update their own profile
  CREATE POLICY "Users can update own profile"
  ON users FOR UPDATE
  USING (auth.uid() = id);
  
  -- Only authenticated users can insert (via onboarding)
  CREATE POLICY "Authenticated users can create profile"
  ON users FOR INSERT
  WITH CHECK (auth.role() = 'authenticated');
  ```
</Accordion>

### stories

<Accordion title="Schema" icon="book">
  ```sql
  CREATE TABLE stories (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    author_wallet TEXT NOT NULL REFERENCES users(wallet_address),
    title TEXT NOT NULL,
    content TEXT NOT NULL,
    mood TEXT,
    tags TEXT[] DEFAULT '{}',
    audio_url TEXT,
    is_public BOOLEAN DEFAULT true,
    is_canonical BOOLEAN DEFAULT false,
    view_count INTEGER DEFAULT 0,
    like_count INTEGER DEFAULT 0,
    comment_count INTEGER DEFAULT 0,
    paywall_price DECIMAL(18, 2) DEFAULT 0,
    ipfs_hash TEXT,
    story_date DATE DEFAULT CURRENT_DATE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
  );
  
  CREATE INDEX idx_stories_author ON stories(author_wallet);
  CREATE INDEX idx_stories_public ON stories(is_public);
  CREATE INDEX idx_stories_date ON stories(story_date DESC);
  CREATE INDEX idx_stories_tags ON stories USING GIN(tags);
  CREATE INDEX idx_stories_created ON stories(created_at DESC);
  ```
  
  **Key Fields:**
  - `is_canonical` — User-marked as important ("Key Moment")
  - `paywall_price` — Price to unlock (0 = free)
  - `ipfs_hash` — IPFS CID if uploaded
</Accordion>

<Accordion title="RLS Policies" icon="shield">
  ```sql
  -- Public stories are viewable by everyone
  CREATE POLICY "Public stories are viewable"
  ON stories FOR SELECT
  USING (is_public = true OR auth.uid() = author_wallet);
  
  -- Only author can update/delete
  CREATE POLICY "Authors can modify own stories"
  ON stories FOR ALL
  USING (auth.uid() = author_wallet);
  
  -- Authenticated users can create stories
  CREATE POLICY "Authenticated users can create stories"
  ON stories FOR INSERT
  WITH CHECK (auth.role() = 'authenticated' AND auth.uid() = author_wallet);
  ```
</Accordion>

### story_metadata

<Accordion title="Schema" icon="sparkles">
  ```sql
  CREATE TABLE story_metadata (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    story_id UUID UNIQUE NOT NULL REFERENCES stories(id) ON DELETE CASCADE,
    
    -- AI-extracted fields
    themes TEXT[] DEFAULT '{}',
    emotional_tone TEXT, -- 'reflective', 'joyful', 'anxious', 'hopeful', etc.
    life_domain TEXT,    -- 'work', 'relationships', 'health', 'identity', 'growth'
    
    -- Scoring (Self-Defining Memory detection)
    intensity_score FLOAT DEFAULT 0,     -- 0-1, emotional charge
    significance_score FLOAT DEFAULT 0,  -- 0-1, life event importance
    
    -- User controls
    is_canonical BOOLEAN DEFAULT false,  -- User-marked as important
    ai_readable BOOLEAN DEFAULT true,    -- Consent for AI analysis
    
    -- Extracted entities
    people_mentioned TEXT[] DEFAULT '{}',
    places_mentioned TEXT[] DEFAULT '{}',
    time_references TEXT[] DEFAULT '{}',
    
    -- Brief insight from analysis
    brief_insight TEXT,
    
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
  );
  
  CREATE INDEX idx_story_metadata_story ON story_metadata(story_id);
  CREATE INDEX idx_story_metadata_themes ON story_metadata USING GIN(themes);
  CREATE INDEX idx_story_metadata_domain ON story_metadata(life_domain);
  CREATE INDEX idx_story_metadata_canonical ON story_metadata(is_canonical);
  ```
  
  **AI Analysis Fields:**
  - Generated by `POST /api/ai/analyze`
  - Powered by Google Gemini 2.5 Flash
  - Automatically created on story save
</Accordion>

<Accordion title="RLS Policies" icon="shield">
  ```sql
  -- Metadata inherits story visibility
  CREATE POLICY "Metadata visible if story is visible"
  ON story_metadata FOR SELECT
  USING (
    EXISTS (
      SELECT 1 FROM stories
      WHERE stories.id = story_metadata.story_id
      AND (stories.is_public = true OR auth.uid() = stories.author_wallet)
    )
  );
  
  -- Only story author or system can write metadata
  CREATE POLICY "Story author can manage metadata"
  ON story_metadata FOR ALL
  USING (
    EXISTS (
      SELECT 1 FROM stories
      WHERE stories.id = story_metadata.story_id
      AND auth.uid() = stories.author_wallet
    )
  );
  ```
</Accordion>

### follows

<Accordion title="Schema" icon="user-plus">
  ```sql
  CREATE TABLE follows (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  ```

### nonces

<Accordion title="Schema" icon="key">
  ```sql
  CREATE TABLE nonces (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    wallet_address TEXT NOT NULL,
    nonce TEXT NOT NULL,
    expires_at TIMESTAMP WITH TIME ZONE NOT NULL,
    used BOOLEAN DEFAULT false,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
  );
  
  CREATE INDEX idx_nonces_wallet ON nonces(wallet_address);
  CREATE INDEX idx_nonces_nonce ON nonces(nonce);
  CREATE INDEX idx_nonces_expires ON nonces(expires_at);
  ```
  
  **Purpose:** Prevents signature replay attacks in wallet authentication
  
  **Lifecycle:**
  1. Generated by `GET /api/auth/nonce?address=0x...`
  2. Single-use (marked `used: true` after verification)
  3. Expires after 5 minutes
</Accordion>

## Verification Tables (CRE)

### verified_metrics

<Accordion title="Schema" icon="shield-check">
  ```sql
  CREATE TABLE verified_metrics (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    story_id UUID NOT NULL REFERENCES stories(id) ON DELETE CASCADE,
    significance_score INTEGER DEFAULT 0,      -- 0-100
    emotional_depth INTEGER DEFAULT 1,          -- 1-5
    quality_score INTEGER DEFAULT 0,            -- 0-100
    word_count INTEGER DEFAULT 0,
    verified_themes TEXT[] DEFAULT '{}',
    metrics_hash TEXT,                          -- keccak256 of all metrics + salt
    quality_tier INTEGER DEFAULT 1,             -- 1-5 derived from quality_score
    meets_quality_threshold BOOLEAN DEFAULT FALSE,
    cre_attestation_id TEXT,
    on_chain_tx_hash TEXT,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    UNIQUE(story_id)
  );
  
  CREATE INDEX idx_verified_metrics_story ON verified_metrics(story_id);
  CREATE INDEX idx_verified_metrics_tier ON verified_metrics(quality_tier DESC);
  ```
  
  **Privacy Model:**
  - Full metrics stored off-chain (Supabase)
  - Only tier, threshold, and hashes on-chain (PrivateVerifiedMetrics.sol)
  - Author-only access via `POST /api/cre/check`
  
  **Integration:**
  - Triggered by `POST /api/cre/trigger`
  - Callback from DON to `POST /api/cre/callback`
  - Read via `useVerifiedMetrics` hook
</Accordion>

## Local Vault (IndexedDB)

### Database: estory-vault

Client-side encrypted storage using Dexie.js. All text fields are AES-256-GCM encrypted.

<Warning>
  This is NOT a Supabase table. It lives entirely in the browser via IndexedDB.
</Warning>

### stories (Vault)

<Accordion title="Schema" icon="lock">
  ```typescript
  interface LocalStoryRecord {
    localId: string;              // UUID, primary key
    userId: string;               // Owner's user ID (indexed)
    encrypted_title: string;      // Base64 AES-GCM ciphertext
    title_iv: string;             // Base64 IV for title
    encrypted_content: string;    // Base64 AES-GCM ciphertext
    content_iv: string;           // Base64 IV for content
    encrypted_audio?: string;     // Base64 encrypted audio blob
    audio_iv?: string;            // Base64 IV for audio
    encrypted_metadata?: string;  // Base64 encrypted JSON (mood, tags)
    metadata_iv?: string;         // Base64 IV for metadata
    checksum: string;             // SHA-256 hash of plaintext for integrity
    cloud_id?: string;            // Cloud story ID after sync (indexed)
    sync_status: SyncStatus;      // 'local' | 'pending' | 'synced' | 'error' (indexed)
    is_public: boolean;           // Whether story is public
    story_date: string;           // User-selected story date
    created_at: string;           // ISO timestamp
    updated_at: string;           // ISO timestamp (indexed)
  }
  
  type SyncStatus = "local" | "pending" | "synced" | "error";
  ```
  
  **Implementation:** `lib/vault/db.ts`
</Accordion>

### vaultKeys

<Accordion title="Schema" icon="key">
  ```typescript
  interface VaultKeyRecord {
    userId: string;        // Owner's user ID (primary key)
    salt: string;          // Base64 PBKDF2 salt (16 bytes)
    wrapped_dek: string;   // Base64 AES-KW wrapped DEK
    pin_hash: string;      // SHA-256 hash of PIN+salt for quick verification
    created_at: string;    // ISO timestamp
  }
  ```
  
  **Encryption Flow:**
  1. User enters 6-digit PIN
  2. PIN → PBKDF2 (100K iterations) → KEK
  3. KEK unwraps DEK via AES-KW
  4. DEK decrypts story fields via AES-256-GCM
</Accordion>

### syncQueue

<Accordion title="Schema" icon="cloud-upload">
  ```typescript
  interface SyncQueueRecord {
    id?: number;                  // Auto-incremented primary key
    storyLocalId: string;         // References stories.localId (indexed)
    userId: string;               // Owner's user ID (indexed)
    action: "upload" | "update" | "delete";
    status: "pending" | "processing" | "failed" (indexed);
    retryCount: number;
    lastAttempt?: string;         // ISO timestamp of last attempt
    error?: string;               // Last error message
    created_at: string;           // ISO timestamp
  }
  ```
  
  **Purpose:** Queue for syncing local vault stories to Supabase cloud
</Accordion>

## Database Constraints

### Foreign Key Cascades

<Note>
  All foreign keys use `ON DELETE CASCADE` to maintain referential integrity.
</Note>

| Table | Foreign Key | On Delete |
|-------|-------------|----------|
| `stories` | `author_wallet` → `users.wallet_address` | CASCADE |
| `story_metadata` | `story_id` → `stories.id` | CASCADE |
| `likes` | `story_id` → `stories.id` | CASCADE |
| `comments` | `story_id` → `stories.id` | CASCADE |
| `saved_stories` | `story_id` → `stories.id` | CASCADE |
| `books` | `author_wallet` → `users.wallet_address` | CASCADE |
| `notifications` | `user_wallet` → `users.wallet_address` | CASCADE |
| `habits` | `user_wallet` → `users.wallet_address` | CASCADE |
| `weekly_reflections` | `user_wallet` → `users.wallet_address` | CASCADE |
| `verified_metrics` | `story_id` → `stories.id` | CASCADE |
| `verification_logs` | `story_id` → `stories.id` | CASCADE |

### Unique Constraints

| Table | Constraint | Purpose |
|-------|-----------|----------|
| `users` | `(wallet_address)` | One account per wallet |
| `users` | `(google_id)` | One account per Google ID |
| `users` | `(username)` | Unique usernames |
| `story_metadata` | `(story_id)` | One metadata record per story |
| `likes` | `(story_id, user_wallet)` | One like per user per story |
| `follows` | `(follower_wallet, following_wallet)` | One follow relationship |
| `saved_stories` | `(story_id, user_wallet)` | One bookmark per user per story |
| `verified_metrics` | `(story_id)` | One CRE verification per story |
