### Client Variants

| File | Use Case | Auth Method |
|------|----------|-------------|
| `app/utils/supabase/supabaseClient.ts` | Browser/client components | User session (RLS enforced) |
| `app/utils/supabase/supabaseServer.ts` | API routes, SSR | Cookies (RLS enforced) |
| `app/utils/supabase/supabaseAdmin.ts` | Admin operations | Service role key (RLS bypassed) |

<Warning>
  **NEVER** initialize Supabase client at module scope in API routes. Use a lazy factory/singleton pattern to avoid build-time failures when env vars are unavailable. See `CLAUDE.md:350-360` for the pattern.
</Warning>

### Local Storage (Client-Side Encryption)

<Accordion title="Dexie.js 4.3.0" icon="hard-drive">
  **Purpose:** IndexedDB wrapper for local encrypted storage
  
  - Database: `estory-vault`
  - Stores encrypted stories with AES-256-GCM
  - Reactive queries via `dexie-react-hooks@4.2.0`
  - Schema: `lib/vault/db.ts`
  - `package.json:62-63`
  
  **Tables:**
  - `stories` — Encrypted story records
  - `vaultKeys` — Wrapped DEKs (Data Encryption Keys)
  - `syncQueue` — Pending cloud sync operations
</Accordion>

<Accordion title="Web Crypto API" icon="lock">
  **Purpose:** Browser-native encryption
  
  - AES-256-GCM for story encryption
  - PBKDF2 (100K iterations) for PIN → KEK derivation
  - AES-KW for DEK wrapping
  - SHA-256 for integrity checksums
  - Implementation: `lib/vault/crypto.ts`, `lib/vault/keyManager.ts`
  
  <Note>
    All crypto operations are browser-only (`window.crypto.subtle`). Never import vault modules in API routes or server components.
  </Note>
</Accordion>

```bash
# Supabase

NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key

# AI Services
GOOGLE_GENERATIVE_AI_API_KEY=your_google_gemini_key
ELEVENLABS_API_KEY=your_elevenlabs_key
ANTHROPIC_API_KEY=your_anthropic_key

# Web3 & Blockchain
NEXT_PUBLIC_PROJECT_ID=your_walletconnect_project_id
NEXT_PUBLIC_ESTORY_TOKEN_ADDRESS=0x...
NEXT_PUBLIC_STORY_PROTOCOL_ADDRESS=0x...
NEXT_PUBLIC_STORY_NFT_ADDRESS=0x...

# Hardhat (deployment only)
PRIVATE_KEY=your_deployer_private_key
BASESCAN_API_KEY=your_basescan_key

# External Services
PINATA_API_KEY=your_pinata_key
PINATA_SECRET_API_KEY=your_pinata_secret
RESEND_API_KEY=your_resend_key

# Security
CRON_SECRET=your_cron_secret
ADMIN_SECRET=your_admin_secret
```

<Warning>
  In production, use Vercel environment variables with scoped access. Never commit `.env.local` to version control.
</Warning>
