## Overview

eStory's API routes are organized by feature domain and follow RESTful conventions. All routes require Bearer token authentication and are protected by rate limiting middleware.

**Location:** `app/api/`

---

## Route Organization

Routes are grouped by feature domain for better maintainability:

```
app/api/
├── ai/                  # AI services (transcription, enhancement, analysis)
├── auth/                # Authentication (wallet + Google OAuth)
├── stories/             # Story CRUD operations
├── social/              # Social features (likes, follows)
├── cre/                 # Chainlink CRE verification
├── ipfs/                # IPFS uploads (via Pinata)
├── email/               # Email sending (via Resend)
├── notifications/       # User notifications
├── user/                # User profile management
├── books/               # Book compilation and minting
├── habits/              # Habit tracking (planned)
├── journal/             # Journal entries (planned)
├── tip/                 # Tipping transactions
├── paywall/             # Paywall management
├── sync/                # Vault sync verification
├── waitlist/            # Waitlist signups
├── audio/               # Audio file uploads
├── cron/                # Scheduled tasks
└── admin/               # Admin-only routes
```

## Route Pattern

Each route folder contains a `route.ts` file that exports HTTP method handlers:

```tsx
// app/api/stories/route.ts
import { NextRequest, NextResponse } from "next/server";
import { validateAuthOrReject, isAuthError } from "@/lib/auth";

export async function GET(req: NextRequest) {
  const authResult = await validateAuthOrReject(req);
  if (isAuthError(authResult)) return authResult;
  const userId = authResult;

  // ... fetch stories for userId

  return NextResponse.json({ stories: data });
}

export async function POST(req: NextRequest) {
  const authResult = await validateAuthOrReject(req);
  if (isAuthError(authResult)) return authResult;
  const userId = authResult;

  const body = await req.json();

  // ... create story

  return NextResponse.json({ story: inserted });
}
```

## Authentication

All API routes (except `/api/auth/*` and `/api/waitlist`) require Bearer token authentication.

### Auth Pattern

<Accordion title="Standard Auth Check">
  ```tsx
  import { validateAuthOrReject, isAuthError } from "@/lib/auth";

  export async function POST(req: NextRequest) {
    // Step 1: Validate token
    const authResult = await validateAuthOrReject(req);
    if (isAuthError(authResult)) return authResult; // Returns 401 if invalid

    // Step 2: Extract user ID
    const userId = authResult; // JWT user ID (Google or wallet)

    // Step 3: Use userId for operations
    const body = await req.json();
    // ...
  }
  ```
</Accordion>

<Accordion title="validateAuthOrReject Function">
  Located in `lib/auth.ts`, this function:

  1. Extracts Bearer token from `Authorization` header
  2. Tries Supabase JWT verification (Google/OAuth users)
  3. Falls back to custom wallet JWT verification (wallet users)
  4. Returns user ID if valid, or `NextResponse` 401 if invalid

  ```tsx
  // lib/auth.ts
  export async function validateAuthOrReject(
    request: NextRequest
  ): Promise<string | NextResponse> {
    const userId = await validateAuth(request);
    if (!userId) {
      return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
    }
    return userId;
  }
  ```
</Accordion>

### Ownership Verification

For routes that modify resources, verify the user owns the resource:

```tsx
import { validateAuthOrReject, isAuthError, resolveUserId } from "@/lib/auth";

export async function POST(req: NextRequest) {
  const authResult = await validateAuthOrReject(req);
  if (isAuthError(authResult)) return authResult;

  const body = await req.json();
  const { author_id } = body;

  // Resolve JWT user ID → users table ID (wallet users may differ)
  const resolvedUserId = await resolveUserId(authResult);

  // Verify ownership
  if (resolvedUserId !== author_id) {
    return NextResponse.json({ error: "Forbidden" }, { status: 403 });
  }

  // ... proceed with operation
}
```

## AI Routes

### POST /api/ai/transcribe

**File:** `app/api/ai/transcribe/route.ts`

Transcribes audio to text using ElevenLabs Scribe.

<Accordion title="Request">
  ```tsx
  const formData = new FormData();
  formData.append("file", audioBlob);

  const response = await fetch("/api/ai/transcribe", {
    method: "POST",
    headers: {
      Authorization: `Bearer ${token}`,
    },
    body: formData,
  });
  ```
</Accordion>

<Accordion title="Response">
  ```json
  {
    "text": "Today I realized that..."
  }
  ```
</Accordion>

<Accordion title="Validation">
  - Max file size: 25MB
  - Allowed types: audio/webm, audio/wav, audio/mp3, audio/mpeg, audio/ogg, audio/flac
  - Returns 400 if validation fails
  - Returns 422 if transcription is empty
</Accordion>

## Auth Routes

### GET /api/auth/nonce

**File:** `app/api/auth/nonce/route.ts`

Generates a one-time nonce for wallet signature verification.

```tsx
const response = await fetch(`/api/auth/nonce?address=${walletAddress}`);
const { message } = await response.json();
// message: "Sign this message to verify your identity. Nonce: abc123..."
```

**Features:**
- 5-minute expiry
- One-time use (deleted after verification)
- Stores in Supabase `nonces` table

### GET /api/auth/callback

**File:** `app/api/auth/callback/route.ts`

Handles Google OAuth redirect after successful login.

**Flow:**
1. User clicks "Sign in with Google"
2. Redirected to Google OAuth
3. Google redirects back to `/api/auth/callback?code=...`
4. This route validates the code and redirects to home

## CRE Routes (Chainlink Verification)

### POST /api/cre/trigger

**File:** `app/api/cre/trigger/route.ts`

Triggers Chainlink CRE verification workflow for a story.

```tsx
const response = await fetch("/api/cre/trigger", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${token}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({ storyId: "123" }),
});
```

**Returns:**
```json
{
  "success": true,
  "message": "CRE verification workflow triggered"
}
```

---

### POST /api/cre/callback

**File:** `app/api/cre/callback/route.ts`

Receives CRE results from Chainlink DON nodes (called by CRE workflow, not by frontend).

**Request body:**
```json
{
  "storyId": "123",
  "significance_score": 8.5,
  "emotional_depth": 7.2,
  "quality_score": 8.0,
  "word_count": 450,
  "verified_themes": ["growth", "relationships"],
  "qualityTier": 4,
  "meetsQualityThreshold": true,
  "metricsHash": "0xabc...",
  "attestationId": "cre-123",
  "on_chain_tx_hash": "0xdef..."
}
```

**Behavior:**
- Stores full metrics in Supabase (author-only access)
- Dual-write: on-chain proof + off-chain full data

---

### POST /api/cre/check

**File:** `app/api/cre/check/route.ts`

Checks CRE verification status (with privacy filtering).

```tsx
const response = await fetch("/api/cre/check", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${token}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({ storyId: "123" }),
});

const { verified, isAuthor, metrics, proof } = await response.json();
```

**Privacy Model:**
- **Author** - Receives full `metrics` + `proof`
- **Public** - Receives `proof` only (tier, threshold, hash)

## Error Handling

<CardGroup cols={2}>
  <Card title="Standard Errors" icon="exclamation-triangle">
    - `400` - Bad request (validation failure)
    - `401` - Unauthorized (missing/invalid token)
    - `403` - Forbidden (not owner of resource)
    - `404` - Not found
    - `409` - Conflict (duplicate entry)
    - `422` - Unprocessable entity (semantic error)
    - `429` - Too many requests (rate limit)
    - `500` - Internal server error
  </Card>

  <Card title="Error Response Format" icon="code">
    ```json
    {
      "error": "Human-readable message",
      "code": "ERROR_CODE" // optional
    }
    ```

    **Never expose internal error details** (stack traces, database errors, etc.) to the client.
  </Card>
</CardGroup>
