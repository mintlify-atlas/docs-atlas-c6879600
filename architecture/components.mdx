## Overview

eStory's frontend is built with React 19 and Next.js 15, using a modular component architecture that separates UI primitives, custom business logic, and provider context.

## Component Structure

```
components/
├── ui/                    # shadcn/ui primitives
├── vault/                 # Vault-specific components
├── three/                 # Three.js backgrounds
├── patterns/              # Pattern analysis views
├── emails/                # Email templates
└── [Custom Components]    # Business logic components
```

## Custom Components

### AuthProvider

**Location:** `components/AuthProvider.tsx`

Manages authentication state for both wallet (MetaMask) and Google OAuth users.

<Accordion title="Features">
  - Unified profile interface for wallet + Google users
  - Automatic wallet signature flow on connection
  - Google OAuth integration via Supabase
  - Secure token storage (Supabase JWT + wallet JWT)
  - Account linking (connect Google to existing wallet account)
  - Onboarding flow management
</Accordion>

<Accordion title="API">
  ```tsx
  interface AuthContextType {
    profile: UnifiedUserProfile | null;
    isLoading: boolean;
    needsOnboarding: boolean;
    getAccessToken: () => Promise<string | null>;
    signInWithGoogle: () => Promise<void>;
    signOut: () => Promise<void>;
    completeOnboarding: (data: OnboardingData) => Promise<void>;
    refreshProfile: () => Promise<void>;
  }
  ```
</Accordion>

<Accordion title="Usage">
  ```tsx
  import { useAuth } from "@/components/AuthProvider";

  export function MyPage() {
    const { profile, getAccessToken } = useAuth();

    const handleSave = async () => {
      const token = await getAccessToken();
      await fetch("/api/stories", {
        method: "POST",
        headers: {
          Authorization: `Bearer ${token}`,
          "Content-Type": "application/json",
        },
        body: JSON.stringify({ title: "My Story" }),
      });
    };

    return <div>Welcome, {profile?.name}!</div>;
  }
  ```
</Accordion>

### StoryCard

**Location:** `components/StoryCard.tsx`

Displays a story with metadata, actions, and optional paywall.

<Accordion title="Props Interface">
  ```tsx
  interface StoryCardProps {
    story: StoryDataType;
    onLike?: (id: number) => void;
    onFollow?: (username: string) => void;
    onShare?: (id: number) => void;
    onUnlock?: (id: number) => void;
    variant?: 'default' | 'compact' | 'featured';
  }
  ```
</Accordion>

<Accordion title="Features">
  - **Author info** - Avatar, name, username, badges, followers
  - **Metadata** - Date, visibility (public/private), audio indicator
  - **Content** - Title, body, tags, emotional tone
  - **Verified badge** - Chainlink CRE verification status
  - **Canonical indicator** - Star icon for important stories
  - **Paywall support** - Teaser + unlock button for premium content
  - **Actions** - Like, comment, share, tip (with slider)
  - **Emotional tone styling** - Left border color based on tone
</Accordion>

```tsx
// Compact variant for grid views
<StoryCard story={story} variant="compact" />

// Full variant with actions
<StoryCard
  story={story}
  variant="default"
  onLike={handleLike}
  onShare={handleShare}
/>
```

### VaultGuard

**Location:** `components/vault/VaultGuard.tsx`

Protects content that requires an unlocked local vault.

<Accordion title="Props">
  ```tsx
  interface VaultGuardProps {
    children: React.ReactNode;
    optional?: boolean;        // Render children even if vault not set up
    fallback?: React.ReactNode; // Custom UI when vault is locked
  }
  ```
</Accordion>

<Accordion title="Behavior">
  - **Not set up** - Shows setup prompt (or renders children if `optional=true`)
  - **Locked** - Prompts for PIN with unlock modal
  - **Unlocked** - Renders children
  - **Loading** - Shows spinner
</Accordion>

```tsx
import { VaultGuard } from "@/components/vault/VaultGuard";

function LocalStoriesPage() {
  return (
    <VaultGuard>
      {/* Only rendered when vault is unlocked */}
      <LocalStoriesList />
    </VaultGuard>
  );
}
```

## Provider Stack

**Location:** `components/Provider.tsx`

The Provider component wraps the entire app and provides context for Web3, theming, auth, and app state.

### Stack Order (outer to inner)

```tsx
WagmiProvider              // Web3 wallet connection
  → QueryClientProvider    // React Query for data fetching
    → ThemeProvider         // Dark/light mode (next-themes)
      → RainbowKitProvider  // Wallet UI (theme-aware)
        → BackgroundProvider // Three.js background settings
          → AuthProvider    // Authentication state
            → AppContext    // App-wide state (user, balance, tokens)
```

<Accordion title="Provider Stack Source">
  ```tsx components/Provider.tsx
  export function Providers({ children }: { children: React.ReactNode }) {
    return (
      <WagmiProvider config={config}>
        <QueryClientProvider client={queryClient}>
          <ThemeProvider attribute="class" defaultTheme="system" enableSystem>
            <ThemedRainbowKit>
              <BackgroundProvider>
                <AuthProvider>
                  <AppInner>{children}</AppInner>
                </AuthProvider>
              </BackgroundProvider>
            </ThemedRainbowKit>
          </ThemeProvider>
        </QueryClientProvider>
      </WagmiProvider>
    );
  }
  ```
</Accordion>

### AppContext

Provides app-wide user state derived from Wagmi hooks.

```tsx
interface AppContextType {
  user: User | null;
  isConnected: boolean;
  isDisconnected: boolean;
  connectWallet: () => void;
  disconnectWallet: () => void;
}

interface User {
  address: string;
  addressDisplay: string;  // Truncated (0x1234...5678)
  balance: string;         // ETH balance
  storyTokens: number;     // $STORY token balance
  badges: string[];        // Achievement badges
}
```

## Component Best Practices

<CardGroup cols={2}>
  <Card title="Client vs Server" icon="server">
    - Use `"use client"` for interactive components
    - Keep page.tsx as server components
    - Load Web3 hooks only in client components
  </Card>

  <Card title="Performance" icon="gauge">
    - Lazy load Three.js with `dynamic(() => import(), { ssr: false })`
    - Use `Skeleton` components during loading
    - Memoize expensive calculations with `useMemo`
  </Card>

  <Card title="Styling" icon="paintbrush">
    - Use Tailwind utility classes
    - Custom colors via CSS variables (hsl)
    - Dark mode via `dark:` prefix
  </Card>

  <Card title="Accessibility" icon="universal-access">
    - Use semantic HTML
    - Include ARIA labels on interactive elements
    - Ensure keyboard navigation works
  </Card>
</CardGroup>
