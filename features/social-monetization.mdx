## Overview

eStory combines social features with Web3 tokenomics to reward storytellers. The $STORY ERC20 token enables tips, paywalls, and NFT minting, while the StoryProtocol smart contract handles all on-chain transactions. Users can like stories, follow creators, and pay to unlock premium content.

## StoryProtocol Contract

### Contract Architecture

The `StoryProtocol` contract manages tips and paywall payments:

```solidity
contract StoryProtocol is ReentrancyGuard, Pausable, AccessControl {
  using SafeERC20 for IERC20;
  
  IERC20 public immutable storyToken;
  
  event TipSent(address indexed from, address indexed to, uint256 amount, uint256 indexed storyId);
  event ContentUnlocked(address indexed payer, address indexed author, uint256 amount, uint256 indexed contentId);
  
  function tipCreator(address creator, uint256 amount, uint256 storyId) external nonReentrant whenNotPaused;
  function payPaywall(address author, uint256 amount, uint256 contentId) external nonReentrant whenNotPaused;
}
```

**Security features:**
- **ReentrancyGuard** - Prevents reentrancy attacks
- **SafeERC20** - Prevents silent transfer failures
- **Pausable** - Emergency stop mechanism
- **AccessControl** - Role-based admin functions

### Tipping Flow

<Steps>
  <Step title="User Approves Token">
    User calls `approve()` on eStoryToken contract to allow StoryProtocol to spend tokens.
  </Step>
  
  <Step title="Call tipCreator()">
    User calls `tipCreator(creatorAddress, amount, storyId)` on StoryProtocol.
  </Step>
  
  <Step title="Transfer Tokens">
    Contract transfers tokens from tipper to creator via `safeTransferFrom()`.
  </Step>
  
  <Step title="Emit Event">
    `TipSent` event is emitted with sender, receiver, amount, and story ID.
  </Step>
  
  <Step title="Backend Indexes">
    Backend listener reads event, updates Supabase `tips` table, creates notification.
  </Step>
</Steps>

#### Frontend Integration

```typescript
import { useStoryProtocol } from "@/app/hooks/useStoryProtocol";
import { useEStoryToken } from "@/app/hooks/useEStoryToken";
import { parseUnits } from "viem";

const { approve } = useEStoryToken();
const { tipCreator } = useStoryProtocol();

const handleTip = async (creatorAddress: string, storyId: string, amount: string) => {
  const amountWei = parseUnits(amount, 18); // e.g., "10" → 10e18
  
  // 1. Approve StoryProtocol to spend tokens
  await approve({
    spender: STORY_PROTOCOL_ADDRESS,
    amount: amountWei,
  });
  
  // 2. Send tip
  await tipCreator({
    creator: creatorAddress as `0x${string}`,
    amount: amountWei,
    storyId: BigInt(storyId),
  });
  
  toast.success(`Tipped ${amount} STORY!`);
};
```

<Warning>
  **Gas costs**: Each tip requires two transactions (approve + tipCreator). Consider implementing multicall or permit (EIP-2612) for single-transaction tips.
</Warning>

## Social Features

### Likes

Users can like stories via `/api/social/like`:

**Request:**
```typescript
POST /api/social/like
Authorization: Bearer {token}
Content-Type: application/json

{
  "storyId": "123e4567-e89b-12d3-a456-426614174000"
}
```

**Response:**
```json
{
  "success": true,
  "data": {
    "storyId": "123e4567-e89b-12d3-a456-426614174000",
    "isLiked": true,
    "totalLikes": 42,
    "timestamp": "2026-03-03T14:32:18Z"
  },
  "message": "Story liked!"
}
```

**Database operations:**
1. **Toggle like record** - Insert/delete in `likes` table (upsert with conflict handling)
2. **Update count** - Increment/decrement `stories.likes` column
3. **Create notification** - Insert in `notifications` table (if not self-like)

<CodeGroup>
```typescript Like Logic
const { data: existingLike } = await admin
  .from("likes")
  .select("id")
  .eq("user_id", authenticatedUserId)
  .eq("story_id", storyId)
  .maybeSingle();

if (existingLike) {
  // Unlike: delete like record
  await admin.from("likes").delete().eq("id", existingLike.id);
  await admin.from("stories").update({ likes: Math.max(0, currentLikes - 1) }).eq("id", storyId);
  isLiked = false;
} else {
  // Like: insert like record
  await admin.from("likes").upsert({ user_id: authenticatedUserId, story_id: storyId });
  await admin.from("stories").update({ likes: currentLikes + 1 }).eq("id", storyId);
  
  // Create notification
  await admin.from("notifications").insert({
    user_id: storyAuthorId,
    type: "like",
    title: "New Like",
    message: `${likerName} liked your story`,
    story_id: storyId,
  });
  
  isLiked = true;
}
```

```typescript Optimistic UI Update
const [isLiked, setIsLiked] = useState(initialLiked);
const [likeCount, setLikeCount] = useState(initialCount);

const handleLike = async () => {
  // Optimistic update
  setIsLiked(!isLiked);
  setLikeCount((prev) => (isLiked ? prev - 1 : prev + 1));
  
  try {
    const res = await fetch("/api/social/like", {
      method: "POST",
      headers: { "Content-Type": "application/json", Authorization: `Bearer ${token}` },
      body: JSON.stringify({ storyId }),
    });
    
    const { data } = await res.json();
    
    // Sync with server response
    setIsLiked(data.isLiked);
    setLikeCount(data.totalLikes);
  } catch (err) {
    // Rollback on error
    setIsLiked(!isLiked);
    setLikeCount((prev) => (isLiked ? prev + 1 : prev - 1));
    toast.error("Failed to like story");
  }
};
```
</CodeGroup>

### Follows

Users can follow other creators via `/api/social/follow`:

**Request:**
```typescript
POST /api/social/follow
Authorization: Bearer {token}
Content-Type: application/json

{
  "follower_wallet": "0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb",
  "followed_wallet": "0x1234567890abcdef1234567890abcdef12345678"
}
```

**Response:**
```json
{
  "isFollowing": true,
  "followers_count": 156
}
```

**Database operations:**
1. **Ownership check** - Verify authenticated user owns `follower_wallet` (prevents impersonation)
2. **Toggle follow record** - Insert/delete in `follows` table
3. **Update count** - Increment/decrement `users.followers_count`
4. **Create notification** - Insert in `notifications` table (if new follow)

<Info>
  Follows are wallet-based (not user ID-based) to support multi-wallet scenarios and decentralized identity.
</Info>

## Paywall System

### Unlocking Premium Content

Creators can gate stories behind token payments:

<Steps>
  <Step title="Creator Sets Paywall">
    Creator marks story as `paywall: true` and sets `paywall_price` (e.g., 100 STORY).
  </Step>
  
  <Step title="Public Preview">
    Non-paying users see truncated content (first 200 characters) + "Unlock for X STORY" button.
  </Step>
  
  <Step title="User Approves & Pays">
    User approves tokens, calls `payPaywall(authorAddress, amount, contentId)` on StoryProtocol.
  </Step>
  
  <Step title="Backend Indexes Event">
    `ContentUnlocked` event listener updates `paywall_unlocks` table.
  </Step>
  
  <Step title="Full Content Revealed">
    User can now read full story. Unlock persists permanently in database.
  </Step>
</Steps>

### Paywall Enforcement

Backend checks unlock status before returning story content:

```typescript
const { data: unlock } = await supabase
  .from("paywall_unlocks")
  .select("id")
  .eq("story_id", storyId)
  .eq("user_id", authenticatedUserId)
  .maybeSingle();

if (story.paywall && !unlock && story.author_id !== authenticatedUserId) {
  // Return truncated content
  return {
    ...story,
    content: story.content.slice(0, 200) + "...",
    is_locked: true,
    unlock_price: story.paywall_price,
  };
}

// Return full content
return { ...story, is_locked: false };
```

<Warning>
  **Client-side enforcement is NOT secure!** Always validate unlock status on the server before returning full content.
</Warning>

### Revenue Sharing

Paywall payments go directly to authors (100% creator earnings):

```solidity
function payPaywall(address author, uint256 amount, uint256 contentId) external {
  require(amount > 0, "Amount must be > 0");
  storyToken.safeTransferFrom(msg.sender, author, amount); // Direct transfer
  emit ContentUnlocked(msg.sender, author, amount, contentId);
}
```

**No platform fee** - eStory takes 0% of paywall revenue. Future v2 may add optional 5% fee for featured content.

## Notifications System

### Event Triggers

All social actions create real-time notifications:

| Event | Type | Message |
|-------|------|----------|
| Like | `like` | "{name} liked your story" |
| Tip | `tip` | "{name} tipped you {amount} STORY" |
| Follow | `follow` | "{name} started following you" |
| Paywall unlock | `unlock` | "{name} unlocked your story \"{title}\"" |
| Comment (Phase 2) | `comment` | "{name} commented on your story" |

### Notification Schema

```typescript
interface Notification {
  id: string;
  user_id: string;          // Recipient
  type: "like" | "tip" | "follow" | "unlock" | "comment";
  title: string;            // "New Like", "New Tip", etc.
  message: string;          // User-friendly description
  story_id?: string;        // Optional story reference
  read: boolean;            // Read status
  created_at: string;       // ISO 8601
}
```

### Frontend Hook

```typescript
import { useNotifications } from "@/app/hooks/useNotifications";

const { notifications, unreadCount, markAsRead, deleteNotification } = useNotifications();

// Auto-refresh every 30 seconds
useEffect(() => {
  const interval = setInterval(() => {
    notifications.refetch();

  return () => clearInterval(interval);
}, []);
```

### Weekly Contest

Top stories each week win prizes:

**Ranking algorithm:**
```typescript
const score = (
  likes * 1.0 +
  tips_received * 0.5 +
  paywall_unlocks * 2.0 +
  comments * 1.5
) * recency_factor;

// recency_factor = 1.0 for this week, decays to 0.1 over 7 days
const recency_factor = Math.max(0.1, 1 - (daysSincePublished / 7));
```

**Prizes:**
- **1st place**: 1000 STORY + featured NFT mint (free)
- **2nd place**: 500 STORY
- **3rd place**: 250 STORY
- **Top 10**: 100 STORY each
