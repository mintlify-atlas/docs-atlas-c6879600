eStory uses **$STORY**, an ERC20 token deployed on Base, to reward creators and enable monetization. Earn tokens through likes, tips, and paywalled content, then use them to support other creators or trade on DEXs.

## Token Overview

The `EStoryToken` contract is deployed on **Base Sepolia** (testnet):

```solidity
// contracts/iStoryToken.sol
contract EStoryToken is ERC20, ERC20Burnable, ERC20Permit, AccessControl, Pausable {
    uint256 public constant MAX_SUPPLY = 100_000_000 * 10 ** 18; // 100M tokens

    constructor(address initialAdmin)
        ERC20("eStoryToken", "ESTORY")
        ERC20Permit("eStoryToken")
    {
        _grantRole(DEFAULT_ADMIN_ROLE, initialAdmin);
        _grantRole(MINTER_ROLE, initialAdmin);
        _grantRole(PAUSER_ROLE, initialAdmin);

        // Initial supply to admin for liquidity/rewards pool
        _mint(initialAdmin, 1_000_000 * 10 ** decimals());
    }
}
```

## Token Economy

### How Tokens Are Created

1. **Initial Supply:**
   - 1M tokens minted to admin at deployment
   - Used for liquidity pools, rewards, and airdrops

2. **User Rewards (Minted Off-Chain):**
   - Backend calls `mint(address to, uint256 amount)` to reward users
   - Requires `MINTER_ROLE` (backend service account)
   - Subject to `MAX_SUPPLY` cap

```solidity
// contracts/iStoryToken.sol:29-32
function mint(address to, uint256 amount) external onlyRole(MINTER_ROLE) whenNotPaused {
    require(totalSupply() + amount <= MAX_SUPPLY, "Exceeds max supply");
    _mint(to, amount);
}
```

### How Tokens Are Distributed

<Note>
  Token distribution is currently **backend-controlled** via API routes. The backend mints tokens to user wallets based on off-chain activity (likes, streaks, etc.).
</Note>

## Receiving Tips

Users can tip you directly using the `StoryProtocol` contract:

```solidity
// contracts/StoryProtocol.sol (example, not in provided files)
function tipCreator(address creator, uint256 amount) external {
    require(amount > 0, "Amount must be greater than 0");
    eStoryToken.transferFrom(msg.sender, creator, amount);
    emit TipSent(msg.sender, creator, amount);
}
```

**How It Works:**

<Steps>
  <Step title="Reader Views Story">
    A reader finds your story on the social feed or via direct link.
  </Step>

  <Step title="Click Tip Button">
    The story detail page shows a "Tip Author" button (requires wallet connection).
  </Step>

  <Step title="Enter Tip Amount">
    A modal prompts the reader to enter the tip amount in $STORY tokens.
  </Step>

  <Step title="Approve & Send">
    1. **Approve:** Reader approves `StoryProtocol` to spend their $STORY
    2. **Transfer:** `tipCreator` transfers tokens to your wallet
    3. **Event:** `TipSent` event is emitted for tracking

    ```tsx
    // app/hooks/useStoryProtocol.ts (example)
    const { writeContract } = useWriteContract();

    const tipCreator = async (creatorAddress: string, amount: bigint) => {
      await writeContract({
        address: STORY_PROTOCOL_ADDRESS,
        abi: STORY_PROTOCOL_ABI,
        functionName: "tipCreator",
        args: [creatorAddress, amount],
      });
    };
    ```
  </Step>

  <Step title="Receive Tokens">
    Tokens are immediately transferred to your wallet. You can view your balance in the Profile page or on Basescan.
  </Step>
</Steps>

<Note>
  Tips are **direct peer-to-peer transfers**. eStory does not take a platform fee. 100% goes to the creator.
</Note>

## Setting Paywalls

Gate premium stories behind a token paywall:

**Paywall Workflow:**

1. **Author Sets Paywall:**
   - Go to story detail page
   - Click "Set Paywall" (author-only action)
   - Enter token amount (e.g., 100 $STORY)
   - Confirm transaction (stores paywall in `StoryProtocol` contract)

2. **Reader Encounters Paywall:**
   - Story preview shows first 100 characters + "Unlock for X $STORY" button
   - Click "Unlock" to pay

3. **Pay to Unlock:**
   - `payPaywall(storyId, amount)` transfers tokens to author
   - Contract marks story as "unlocked" for reader
   - Full story content is revealed

```solidity
// contracts/StoryProtocol.sol (example)
mapping(bytes32 => uint256) public paywalls; // storyId => amount
mapping(bytes32 => mapping(address => bool)) public hasAccess; // storyId => reader => unlocked

function setPaywall(bytes32 storyId, uint256 amount) external {
    require(amount > 0, "Amount must be greater than 0");
    paywalls[storyId] = amount;
    emit PaywallSet(storyId, msg.sender, amount);
}

function payPaywall(bytes32 storyId, uint256 amount) external {
    require(paywalls[storyId] == amount, "Incorrect amount");
    eStoryToken.transferFrom(msg.sender, ownerOf(storyId), amount);
    hasAccess[storyId][msg.sender] = true;
    emit PaywallPaid(storyId, msg.sender, amount);
}
```

**Checking Access:**

```tsx
// Example frontend logic
const { data: hasAccess } = useReadContract({
  address: STORY_PROTOCOL_ADDRESS,
  abi: STORY_PROTOCOL_ABI,
  functionName: "hasAccess",
  args: [storyId, userAddress],
});

if (!hasAccess && paywallAmount > 0) {
  return <PaywallModal amount={paywallAmount} onPay={handlePay} />;
}
```

<Warning>
  Paywalls are **permanent once paid**. Readers get lifetime access to the story after unlocking. There's no refund mechanism.
</Warning>

## Token Balance

View your $STORY token balance:

### Profile Page

The Profile page shows your balance using the `useEStoryToken` hook:

```tsx
// app/hooks/useEStoryToken.ts (example)
import { useReadContract } from "wagmi";
import { ESTORY_TOKEN_ADDRESS, ESTORY_TOKEN_ABI } from "@/lib/contracts";

export function useEStoryToken() {
  const { address } = useAccount();

  const { data: balance } = useReadContract({
    address: ESTORY_TOKEN_ADDRESS,
    abi: ESTORY_TOKEN_ABI,
    functionName: "balanceOf",
    args: [address],
  });

  return {
    balance: balance ? ethers.utils.formatEther(balance) : "0",
  };
}
```

Permanently remove tokens from circulation:

```solidity
// contracts/iStoryToken.sol (inherited from ERC20Burnable)
function burn(uint256 amount) public virtual {
    _burn(_msgSender(), amount);
```

```tsx

const handleBurn = async (amount: string) => {

    address: ESTORY_TOKEN_ADDRESS,
```

## Token Security

The $STORY token implements several security features:

### Access Control

```solidity
// contracts/iStoryToken.sol:11-12
bytes32 public constant MINTER_ROLE = keccak256("MINTER_ROLE");
bytes32 public constant PAUSER_ROLE = keccak256("PAUSER_ROLE");
```

- **MINTER_ROLE:** Backend service can mint tokens for rewards
- **PAUSER_ROLE:** Admin can pause transfers in emergencies
- **DEFAULT_ADMIN_ROLE:** Contract deployer (can grant/revoke roles)

### Pausable Transfers

```solidity
// contracts/iStoryToken.sol:34-36
function pause() external onlyRole(PAUSER_ROLE) {
    _pause();
}

// contracts/iStoryToken.sol:43-46
function _update(address from, address to, uint256 value) internal override(ERC20) {
    require(!paused(), "Token transfer while paused");
    super._update(from, to, value);
}
```

If a critical vulnerability is found, admins can pause all transfers until fixed.

### ERC20Permit

```solidity
// contracts/iStoryToken.sol:10
import "@openzeppelin/contracts/token/ERC20/extensions/ERC20Permit.sol";
```

Supports gasless approvals via EIP-2612 permit signatures:

```tsx
// Example permit signature
const deadline = Math.floor(Date.now() / 1000) + 3600; // 1 hour
const { v, r, s } = await signTypedData({
  domain: { name: "eStoryToken", chainId: 84532 },
  types: { Permit: [...] },
  message: { owner, spender, value, nonce, deadline },
});

await eStoryToken.permit(owner, spender, value, deadline, v, r, s);
```

This allows users to approve token spending without a separate transaction (saves gas).
